# Porting notes: Fabric 1.16.5 -> 26.2

This mod uses [Stonecutter](https://stonecutter.kikugie.dev/) to build one
source tree into a jar per Minecraft version. `stonecutter.properties.toml`
declares 14 version tiers; `settings.gradle.kts` registers them; the actual
source-level differences are handled with `//? if` comment directives inside
`src/main/java/.../*.java` (see the version-specific `versions/<id>/` trees
generated by Stonecutter for the expanded output per tier).

## Version ladder and why each tier is a separate group

| Tier | MC releases covered | Java | Key API era |
|---|---|---|---|
| 1.16.5 | 1.16-1.16.5 | 8 | pre-GuiGraphics, pre-BuiltInRegistries, pre-Warden |
| 1.17.1 | 1.17-1.17.1 | 16 | pre-GuiGraphics |
| 1.18.2 | 1.18-1.18.2 | 17 | pre-GuiGraphics |
| 1.19.2 | 1.19-1.19.2 | 17 | pre-GuiGraphics, pre-BuiltInRegistries |
| 1.19.4 | 1.19.3-1.19.4 | 17 | pre-GuiGraphics, BuiltInRegistries exists |
| 1.20.1 | 1.20-1.20.1 | 17 | GuiGraphics + legacy blit |
| 1.20.4 | 1.20.2-1.20.4 | 17 | GuiGraphics + legacy blit |
| 1.20.6 | 1.20.5-1.20.6 | 21 | GuiGraphics + legacy blit (Java bump) |
| 1.21.1 | 1.21-1.21.1 | 21 | GuiGraphics + legacy blit |
| 1.21.4 | 1.21.2-1.21.4 | 21 | GuiGraphics + legacy blit |
| 1.21.5 | 1.21.5 | 21 | GuiGraphics + RenderPipelines blit (isolated tier) |
| 1.21.8 | 1.21.6-1.21.8 | 21 | New HUD API (HudElementRegistry) + RenderPipelines blit |
| 26.1.2 | 26.1.x | 25 | New HUD + RenderPipelines + non-obfuscated names |
| 26.2 | 26.2 (active/dev) | 25 | Same as 26.1.2 era |

The boundaries above are where an actual API break happens (HUD rendering
signature, registry class, blit call, or renaming), not arbitrary minor-
version splits. See the inline comment header in `stonecutter.properties.toml`
for the same table plus registry/Warden/rename notes.

## Refactors made to support the full range

- **HUD rendering (`LastHeartFeature.java`, `LastTotemFeature.java`)**: branched
  on the pre/post-1.20 `GuiGraphics` signature, and again on the legacy `blit`
  overload vs the 1.21.5+ `RenderPipelines`-based blit, and again on the
  1.21.8+ `HudElementRegistry` registration API vs the older HUD render event.
- **Registry access (`BossDetector.java`, `HostileStateSignals.java`)**: branched
  on `Registry.ENTITY_TYPE` (<1.19.3) vs `BuiltInRegistries.ENTITY_TYPE`
  (>=1.19.3); Warden references are gated out entirely below 1.19 since the
  entity does not exist yet.
- **Non-obfuscated renames (26.1+)**: `ResourceLocation`/`GuiGraphics` map to
  `Identifier`/`GuiGraphicsExtractor`-style names on the non-obf tiers; handled
  via the Loom back-compat plugin plus `//? if` source branches.
- **Java toolchain**: `build.gradle.kts` and each tier's `fabric.mod.json`
  branch the toolchain/target between Java 8, 16, 17, 21, and 25 as required
  by each Minecraft release.

## Dependency version verification log

All pinned `deps.fabric_api`, `deps.modmenu`, and `deps.cloth_config` values in
`stonecutter.properties.toml` were checked against the live Modrinth API
(`api.modrinth.com/v2/project/{slug}/version`) for the exact `game_versions`
and `fabric` loader filter. Three pins were wrong and have been corrected:

| Tier | Dependency | Old (wrong) pin | Corrected pin | Note |
|---|---|---|---|---|
| 1.17.1 | Cloth Config | `5.0.38` | `5.3.63` | `5.0.38` does not exist for 1.17.1 on Modrinth; only `5.3.63` and `5.3.58+fabric` are published for that game version. |
| 1.20.4 | Cloth Config | `12.0.118+fabric` | `13.0.138+fabric` | `12.0.118+fabric` does not exist; `13.0.138+fabric` is a real published version for 1.20.2-1.20.4. |
| 1.21.5 | Cloth Config | `18.0.146+fabric` | `18.0.145+fabric` | `18.0.146+fabric` does not exist; `18.0.145+fabric` is the real published version for 1.21.5. |

Every other pinned Fabric API, Mod Menu, and Cloth Config version across all
14 tiers was confirmed to exist exactly as published on Modrinth for that
game version and loader. `mod.loader_compat` entries are loose minimum-version
bounds (not exact pins) and were left as-is; any reasonably current Fabric
Loader release satisfies them.
